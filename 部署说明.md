# ws-scrcpy 带声音版 - NAS 部署说明（小白向）

用 **Docker Compose** 在 NAS 上部署支持**播放音频**的 ws-scrcpy，按下面步骤做即可。

---

## 一、你需要准备的东西

1. **NAS**：已安装 Docker，并且能用 Docker Compose（群晖、威联通、绿联等一般都有）。
2. **一整套项目文件**：当前这个文件夹里的**全部内容**（包含 `Dockerfile`、`docker-compose.yml`、`config.yaml`、`_scrcpy_fork` 等），要原样放到 NAS 的某个目录里。

---

## 二、重要：网页要有声音，必须用「带音频的」scrcpy-server

- 本项目**最核心需求**是：**网页投屏并且能听到设备声音**。
- 若 `vendor/Genymobile/scrcpy/scrcpy-server.jar` 是原版（未改过的）jar，则**没有音频**，勾选 Sound 也不会出声。
- 要听到声音，必须使用**本仓库改过的** server（`_scrcpy_fork` 里已实现麦克风采集并发送 `scrcpy_audio_` 协议）。
- **做法**：用「二.五」任一方式拿到带音频的 server 并替换 `vendor/Genymobile/scrcpy/scrcpy-server.jar`，然后再用当前项目构建 Docker 镜像。这样镜像里跑的才是带声音的 server，**不会在修其他问题时把声音修丢**。  
  **若本机没有 Android 环境**：用下面「二.五」里的**做法一（GitHub Actions 云端构建）**即可，无需安装 Android Studio/SDK。

**如何得到带声音的 scrcpy-server.jar**（二.五，任选一种做法）：

1. 在本机安装 **Android SDK**（或 Android Studio），并确保能使用 `adb`、`gradle`。
2. 在项目根目录打开终端，进入 server 目录并构建：
   ```bash
   cd _scrcpy_fork/server
   ./gradlew assembleRelease
   ```
   （若没有 `gradlew`，可先在同一目录执行 `gradle wrapper` 生成，或用系统安装的 gradle：`gradle assembleRelease`。）
3. 构建成功后，在 `_scrcpy_fork/server/build/outputs/apk/release/` 下会生成 **release 的 apk**。  
   将该 apk **复制并改名为** `scrcpy-server.jar`，放到项目的 `vendor/Genymobile/scrcpy/` 目录下，**覆盖**原来的 `scrcpy-server.jar`。
4. 之后按下面步骤在 NAS 上构建并运行 Docker 时，用的就是**带声音的** server；无需再改 Dockerfile 或其它配置。

---

## 三、把项目放到 NAS 上

任选一种方式，只要最后 NAS 上有一个文件夹，里面有这些内容就行：

- `Dockerfile`
- `docker-compose.yml`
- `config.yaml`
- `启动.sh`（没有 Compose 时用这个）
- `_scrcpy_fork` 文件夹（整个目录）
- 以及本项目的其他所有文件（如 `package.json`、`src`、`vendor` 等）

**方式 A：用 U 盘 / 局域网拷贝**  
在你这台电脑上，把整个项目文件夹（例如 `dockerfile` 或 `ws-scrcpy`）复制到 U 盘，再插到 NAS 上拷贝到 NAS 的某个目录；  
或者通过 NAS 的“文件管理 / 共享文件夹”从电脑复制进去。

**方式 B：先打包再上传**  
在电脑上把整个项目文件夹打成 zip，用 NAS 的“文件管理”上传并解压到某个目录。

**方式 C：NAS 支持 Git 的话**  
如果你已经把这个项目放到 Git 仓库里，可以在 NAS 上 clone 到某个目录（要包含 `_scrcpy_fork` 和所有文件）。

## 二.五、如何得到带声音的 scrcpy-server.jar（重要：网页要听到声音必做）

**为什么需要这一步**：项目里的 `vendor/Genymobile/scrcpy/scrcpy-server.jar` 如果是原版（无音频），则投屏时勾选 Sound 也不会有声音。**本项目的核心需求是「网页能听到手机声音」**，因此需要把该 jar 替换成「带音频的 server」再构建 Docker。

**做法一：没有 Android 环境时（推荐）—— 用 GitHub 自动构建并下载**

1. 把本项目放到 **GitHub**（新建仓库后 push 上去，或已有仓库即可）。
2. 在浏览器打开仓库 → 顶部点 **「Actions」** → 左侧选 **「构建带音频的 scrcpy-server」**。
3. 右侧点 **「Run workflow」** → 再点绿色的 **「Run workflow」**，等约 5～15 分钟。
4. 运行完成后，同一页下方 **「Artifacts」** 里会出现 **scrcpy-server-with-audio**，点它下载（得到一个 zip，解压后是 `scrcpy-server-with-audio.apk`）。
5. 把该 **apk 改名为 `scrcpy-server.jar`**，放到你本机项目里的 **`vendor/Genymobile/scrcpy/`** 目录下，覆盖原文件（若有）。
6. 之后按「四、在 NAS 上执行部署」构建并启动即可，**无需本机安装 Android 或 Gradle**。

**做法二：有 Android 开发环境时 —— 本机构建**

1. 安装 Android SDK（或 Android Studio），并确保命令行能用到 `adb` 和 Gradle（或本机已装 Gradle）。
2. 在项目根目录（和 `_scrcpy_fork` 同级）进入 server 目录并构建：
   ```bash
   cd _scrcpy_fork/server
   gradle assembleRelease
   ```
   （若没有 `gradle` 命令，可用 Android Studio 打开 `_scrcpy_fork` 根目录或 server 目录，用菜单 Build → Build Bundle(s) / APK(s) → Build APK(s)，再在 `server/build/outputs/apk/release/` 下找到生成的 apk。）
3. 将生成的 **APK** 复制为项目里的 **scrcpy-server.jar**（Android 上可把 apk 当 classpath 用）。**当前目录是 `_scrcpy_fork/server` 时**执行：
   ```bash
   cp build/outputs/apk/release/server-release.apk ../../vendor/Genymobile/scrcpy/scrcpy-server.jar
   ```
   （若在**项目根目录**执行，则用：`cp _scrcpy_fork/server/build/outputs/apk/release/server-release.apk vendor/Genymobile/scrcpy/scrcpy-server.jar`）
4. 之后按「四、在 NAS 上执行部署」正常构建并启动。**这样镜像里用的就是带音频的 server，网页勾选 Sound 即可听到声音。**

**注意**：后续任何修改（修构建、修连接、修 Docker 等）都要**保留「使用带音频的 server」**，不要把声音需求做丢。

---

## 四、在 NAS 上执行部署

1. **打开 NAS 的终端（SSH）或“任务计划”里的“用户自定义脚本”**  
   - 群晖：控制面板 → 终端机和 SNMP → 启动 SSH，然后用电脑 SSH 登录到 NAS。  
   - 威联通等：在应用里找到“终端”或 SSH 方式登录。

2. **进入你放项目的那个目录**，例如：
   ```bash
   cd /volume1/docker/ws-scrcpy
   ```
   （路径按你实际放的改，例如 `/share/CACHEDEV1_DATA/Container/ws-scrcpy`）

3. **确认这几样都在当前目录**：
   ```bash
   ls -la
   ```
   要能看到：`Dockerfile`、`config.yaml`、`启动.sh`，以及 `_scrcpy_fork` 目录。

4. **构建并启动**（二选一）：

   **【推荐】没有 Docker Compose 时（报错 unknown flag: --detach 或 command not found: docker-compose）**  
   你的 NAS 只有 Docker、没有 Compose，用脚本即可（整行复制）：
   ```bash
   bash 启动.sh
   ```
   若 NAS 是 **Intel/AMD（x86）**，先打开 `启动.sh`，把里面第一段里的 `PLATFORM=linux/arm64` 改成 `PLATFORM=linux/amd64`，保存后再执行上面命令。

   **有 Docker Compose 时**（仅当 `docker compose version` 能正常显示版本再用）：
   ```bash
   docker compose up --detach --build
   ```
   或：`docker-compose up -d --build`

   第一次会下载镜像、编译 Android 和前端，可能要 **20～40 分钟**（视 NAS 性能而定），请耐心等。

5. 看到「完成。浏览器访问…」或类似无报错提示，就说明已经部署好了。

---

## 五、怎么使用

1. **在浏览器里打开**：  
   `http://你的NAS的IP:8000`  
   例如：`http://192.168.1.100:8000`

2. **手机连到同一局域网**，并开启 **USB 调试**（开发者选项里）。

3. **用数据线把手机插到 NAS 的 USB 口**（如果 NAS 有 USB 且 Docker 能访问 USB 设备）。  
   若你的 NAS 没有 USB 或不能直连，需要在一台和 NAS 同局域网的电脑上运行 `adb`，用 `adb connect 手机IP` 等方式连接，具体要看你的网络拓扑。

4. 在网页里选择设备、开始投屏后，在页面工具栏勾选 **「Sound」**，即可听到设备**播放的声音**。  
   **重要**：只有使用**带音频的 scrcpy-server** 时才有声音；若你用的是项目里自带的原版 jar，则无声音。请务必先按**上面「二.五、如何得到带声音的 scrcpy-server.jar」**做一次，再构建镜像，这样网页才能听到声音。

## 六、常用命令（可选）

- **查看是否在运行**：  
  `docker compose ps`

- **查看日志**：  
  `docker compose logs -f`

- **停止**：  
  `docker compose down`

- **以后只更新配置再启动**（不重新构建）：  
  `docker compose up -d`

## 七、可能遇到的问题

| 情况 | 建议 |
|------|------|
| 输入构建命令后报错 | 看下面「报错时怎么办」。 |
| **报错 `unknown flag: --detach` 或 `docker compose` 没有 compose 子命令** | 说明 NAS 上**没有安装 Docker Compose**，请用**不用 Compose**的方式：在项目目录执行 `bash 启动.sh`（见上面「构建并启动」）。 |
| 报错 `unknown shorthand flag: 'd' in -d` | 用 `--detach` 代替 `-d`，或直接改用 `bash 启动.sh`。 |
| NAS 是 Intel/AMD（x86） | 在 `docker-compose.yml` 里把两处 `linux/arm64` 改成 `linux/amd64`，再构建。 |
| **报错 `TLS handshake timeout` 或拉取镜像超时** | 国内网络连 Docker 官方仓库容易超时，需要配置**镜像加速**，见项目里的 **`Docker镜像加速.md`**。 |
| 构建时卡在下载 Android 相关 | 多等一会，或检查 NAS 能否正常上网。 |
| 手机在网页里看不到 | 确认 USB 调试已开、数据线连接正常；若用 adb 网络连接，需在能运行 adb 的电脑上先 `adb connect`。 |
| 勾选 Sound 没声音 | **先确认**已按「二.五」用带音频的 server 替换 `vendor/Genymobile/scrcpy/scrcpy-server.jar` 并重新构建镜像。再检查设备尽量 Android 13+、浏览器未静音、系统音量已开。 |
| 端口 8000 被占用 | 在 `docker-compose.yml` 里把 `"8000:8000"` 改成例如 `"8001:8000"`，然后访问 `http://NAS的IP:8001`。 |

---

## 八、报错时怎么办

1. **把完整报错发出来**：从你输入 `docker compose up -d --build`（或 `docker-compose up -d --build`）开始，到终端里出现的**全部文字**复制下来（可以复制到记事本再发），别人才能根据报错帮你改。

2. **常见报错**：
   - **`config.yaml` 不存在**：确认项目目录里有 `config.yaml` 文件（和 `docker-compose.yml` 同一层）。
   - **`No such file or directory: '_scrcpy_fork'`**：说明项目不完整，要把带 `_scrcpy_fork` 文件夹的整份项目重新拷到 NAS。
   - **`no matching manifest for linux/arm64`**：你的 NAS 可能是 x86，把 `docker-compose.yml` 里的 `linux/arm64` 改成 `linux/amd64`。

按上面步骤做完，你就已经用 docker-compose 把“支持音频的 ws-scrcpy”部署在 NAS 上了。
