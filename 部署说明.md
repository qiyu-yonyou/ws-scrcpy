# ws-scrcpy 带声音版 - NAS 部署说明（小白向）

用 **Docker Compose** 在 NAS 上部署支持**播放音频**的 ws-scrcpy，按下面步骤做即可。

---

## 一、你需要准备的东西

1. **NAS**：已安装 Docker，并且能用 Docker Compose（群晖、威联通、绿联等一般都有）。
2. **一整套项目文件**：当前这个文件夹里的**全部内容**（包含 `Dockerfile`、`docker-compose.yml`、`config.yaml`、`_scrcpy_fork` 等），要原样放到 NAS 的某个目录里。

---

## 二、把项目放到 NAS 上

任选一种方式，只要最后 NAS 上有一个文件夹，里面有这些内容就行：

- `Dockerfile`
- `docker-compose.yml`
- `config.yaml`
- `启动.sh`（没有 Compose 时用这个）
- `_scrcpy_fork` 文件夹（整个目录）
- 以及本项目的其他所有文件（如 `package.json`、`src`、`vendor` 等）

**方式 A：用 U 盘 / 局域网拷贝**  
在你这台电脑上，把整个项目文件夹（例如 `dockerfile` 或 `ws-scrcpy`）复制到 U 盘，再插到 NAS 上拷贝到 NAS 的某个目录；  
或者通过 NAS 的“文件管理 / 共享文件夹”从电脑复制进去。

**方式 B：先打包再上传**  
在电脑上把整个项目文件夹打成 zip，用 NAS 的“文件管理”上传并解压到某个目录。

**方式 C：NAS 支持 Git 的话**  
如果你已经把这个项目放到 Git 仓库里，可以在 NAS 上 clone 到某个目录（要包含 `_scrcpy_fork` 和所有文件）。

---

## 三、在 NAS 上执行部署

1. **打开 NAS 的终端（SSH）或“任务计划”里的“用户自定义脚本”**  
   - 群晖：控制面板 → 终端机和 SNMP → 启动 SSH，然后用电脑 SSH 登录到 NAS。  
   - 威联通等：在应用里找到“终端”或 SSH 方式登录。

2. **进入你放项目的那个目录**，例如：
   ```bash
   cd /volume1/docker/ws-scrcpy
   ```
   （路径按你实际放的改，例如 `/share/CACHEDEV1_DATA/Container/ws-scrcpy`）

3. **确认这几样都在当前目录**：
   ```bash
   ls -la
   ```
   要能看到：`Dockerfile`、`config.yaml`、`启动.sh`，以及 `_scrcpy_fork` 目录。

4. **构建并启动**（二选一）：

   **【推荐】没有 Docker Compose 时（报错 unknown flag: --detach 或 command not found: docker-compose）**  
   你的 NAS 只有 Docker、没有 Compose，用脚本即可（整行复制）：
   ```bash
   bash 启动.sh
   ```
   若 NAS 是 **Intel/AMD（x86）**，先打开 `启动.sh`，把里面第一段里的 `PLATFORM=linux/arm64` 改成 `PLATFORM=linux/amd64`，保存后再执行上面命令。

   **有 Docker Compose 时**（仅当 `docker compose version` 能正常显示版本再用）：
   ```bash
   docker compose up --detach --build
   ```
   或：`docker-compose up -d --build`

   第一次会下载镜像、编译 Android 和前端，可能要 **20～40 分钟**（视 NAS 性能而定），请耐心等。

5. 看到「完成。浏览器访问…」或类似无报错提示，就说明已经部署好了。

---

## 四、怎么使用

1. **在浏览器里打开**：  
   `http://你的NAS的IP:8000`  
   例如：`http://192.168.1.100:8000`

2. **手机连到同一局域网**，并开启 **USB 调试**（开发者选项里）。

3. **用数据线把手机插到 NAS 的 USB 口**（如果 NAS 有 USB 且 Docker 能访问 USB 设备）。  
   若你的 NAS 没有 USB 或不能直连，需要在一台和 NAS 同局域网的电脑上运行 `adb`，用 `adb connect 手机IP` 等方式连接，具体要看你的网络拓扑。

4. 在网页里选择设备、开始投屏后，在页面工具栏勾选 **「Sound」**，即可听到设备**播放的声音**（需 Android 13+ 效果最佳）。

---

## 五、常用命令（可选）

- **查看是否在运行**：  
  `docker compose ps`

- **查看日志**：  
  `docker compose logs -f`

- **停止**：  
  `docker compose down`

- **以后只更新配置再启动**（不重新构建）：  
  `docker compose up -d`

---

## 六、可能遇到的问题

| 情况 | 建议 |
|------|------|
| 输入构建命令后报错 | 看下面「报错时怎么办」。 |
| **报错 `unknown flag: --detach` 或 `docker compose` 没有 compose 子命令** | 说明 NAS 上**没有安装 Docker Compose**，请用**不用 Compose**的方式：在项目目录执行 `bash 启动.sh`（见上面「构建并启动」）。 |
| 报错 `unknown shorthand flag: 'd' in -d` | 用 `--detach` 代替 `-d`，或直接改用 `bash 启动.sh`。 |
| NAS 是 Intel/AMD（x86） | 在 `docker-compose.yml` 里把两处 `linux/arm64` 改成 `linux/amd64`，再构建。 |
| **报错 `TLS handshake timeout` 或拉取镜像超时** | 国内网络连 Docker 官方仓库容易超时，需要配置**镜像加速**，见项目里的 **`Docker镜像加速.md`**。 |
| 构建时卡在下载 Android 相关 | 多等一会，或检查 NAS 能否正常上网。 |
| 手机在网页里看不到 | 确认 USB 调试已开、数据线连接正常；若用 adb 网络连接，需在能运行 adb 的电脑上先 `adb connect`。 |
| 勾选 Sound 没声音 | 设备尽量是 Android 13+；确认浏览器没有静音、系统音量已开。 |
| 端口 8000 被占用 | 在 `docker-compose.yml` 里把 `"8000:8000"` 改成例如 `"8001:8000"`，然后访问 `http://NAS的IP:8001`。 |

---

## 七、报错时怎么办

1. **把完整报错发出来**：从你输入 `docker compose up -d --build`（或 `docker-compose up -d --build`）开始，到终端里出现的**全部文字**复制下来（可以复制到记事本再发），别人才能根据报错帮你改。

2. **常见报错**：
   - **`config.yaml` 不存在**：确认项目目录里有 `config.yaml` 文件（和 `docker-compose.yml` 同一层）。
   - **`No such file or directory: '_scrcpy_fork'`**：说明项目不完整，要把带 `_scrcpy_fork` 文件夹的整份项目重新拷到 NAS。
   - **`no matching manifest for linux/arm64`**：你的 NAS 可能是 x86，把 `docker-compose.yml` 里的 `linux/arm64` 改成 `linux/amd64`。

按上面步骤做完，你就已经用 docker-compose 把“支持音频的 ws-scrcpy”部署在 NAS 上了。
